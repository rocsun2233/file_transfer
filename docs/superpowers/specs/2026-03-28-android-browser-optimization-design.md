# Android 浏览器接入优化设计

日期：2026-03-28

## 1. 概述

当前项目已经具备浏览器受限访问能力，但这套 Web 入口仍然偏桌面化，尚未针对 Android 手机浏览器的小屏和触控行为进行优化。下一轮需要在现有本地 Web 入口基础上，补齐 Android 浏览器访问状态、移动端布局、上传下载入口、最近任务展示和移动端错误反馈，使手机通过浏览器接入时真正可用。

本轮只做 Android 浏览器接入优化，不进入 Android 原生客户端或 PWA 范围。

## 2. 目标与非目标

### 目标

- 让 Android 浏览器能清晰看到当前访问状态
- 提供触控友好的上传、下载和刷新入口
- 提供适合小屏幕的文件与最近任务展示
- 提供明确的权限、连接和操作错误反馈
- 保持与现有桌面浏览器入口共用同一套后端逻辑

### 非目标

- 本轮不实现 Android 原生客户端
- 本轮不实现 PWA 安装能力
- 本轮不做离线支持
- 本轮不做复杂任务管理
- 本轮不做推送通知

## 3. 页面结构

移动优化后的页面分为四个区域。

### 3.1 访问状态区

页面顶部首先展示当前状态：

- 已授权访问
- 等待桌面端确认
- 需要输入一次性验证码
- 拒绝访问或会话失效

### 3.2 快捷操作区

提供移动端最核心的操作：

- 上传文件
- 下载当前可下载项
- 刷新页面
- 查看最近任务

### 3.3 文件与任务列表区

使用适合触控的卡片或大行高列表展示：

- 当前可浏览内容
- 最近任务摘要

避免密集表格和过小点击区域。

### 3.4 连接说明区

展示简短说明：

- 当前连接的设备
- 当前访问限制
- Android 当前通过浏览器接入而非原生客户端

## 4. 模块边界

### 4.1 MobileWebViewModel

负责把权限状态、最近任务、操作入口和移动端提示整理为适合小屏展示的视图模型。

### 4.2 GuestAccessController

继续负责权限判断和会话授权，不在本轮改变其核心职责。

### 4.3 LocalWebGatewayServer

继续负责 HTTP 路由，但需要支持移动优化视图与状态化响应。

## 5. 交互流

### 5.1 访问进入流程

Android 浏览器打开页面后，优先展示访问状态，而不是直接展示文件列表。若未获准访问，则先引导用户等待确认或输入验证码。

### 5.2 上传流程

移动端主按钮直接触发文件选择器，尽量减少中间步骤。上传成功或失败后，页面展示明确结果。

### 5.3 下载流程

对当前允许下载的项目提供直接下载入口，避免多层跳转。

### 5.4 最近任务展示

将最近任务以只读方式展示给手机用户，帮助其快速确认最近上传或下载结果。

## 6. 错误处理

本轮要显式处理以下状态：

- 未授权访问
- 等待桌面端确认
- 验证码错误
- 会话过期
- 上传失败
- 下载失败
- 服务不可达

错误信息应使用短句和明显状态块，而不是返回通用错误页。

## 7. 测试策略

### 7.1 视图模型测试

验证不同权限状态下的移动视图数据是否正确。

### 7.2 接口测试

验证 Android 浏览器访问时的：

- 状态页
- 验证码流程
- 上传下载入口
- 最近任务展示

### 7.3 回归测试

验证移动优化不会破坏现有桌面浏览器访问和桌面主链路行为。

## 8. 技术约束

本轮继续沿用现有本地 Web 入口，不新建独立移动站点，不引入新的前端框架。重点是通过服务端输出和轻量页面结构调整，让 Android 浏览器访问变得清晰可用。

## 9. 后续衔接

本轮完成后，后续可以继续推进：

- Android 更强的浏览器体验
