## Why

现有局域网文件共享工具通常只覆盖单一交互形态，要么要求双方都安装客户端，要么只能通过浏览器访问，难以同时兼顾 Windows、Linux、macOS 的稳定传输体验和临时访问便利性。现在需要定义一个新的跨平台方案，既支持桌面客户端间的可靠文件传输，也支持未安装客户端设备通过浏览器临时参与，并为后续互联网穿透保留明确扩展边界。

## What Changes

- 新增跨平台混合式文件传输应用方案，覆盖 Windows、Linux、macOS。
- 新增桌面主应用与本地核心服务分层架构，统一处理设备发现、配对、任务管理和传输会话。
- 新增局域网设备自动发现、首次连接确认或配对码、持久信任关系能力。
- 新增文件与文件夹传输、批量任务、断点续传、历史记录和拖拽发送能力。
- 新增内置本地 Web 页面，支持浏览器临时访问、受限上传和下载。
- 新增面向未来互联网穿透的连接抽象，允许后续扩展云端目录、NAT 打洞和中继传输。

## Capabilities

### New Capabilities
- `device-discovery-and-trust`: 定义局域网设备发现、首次配对确认、配对码和信任关系管理行为。
- `hybrid-file-transfer`: 定义桌面客户端主导的文件和文件夹传输、批量任务、进度、历史记录与断点续传行为。
- `web-guest-access`: 定义内置本地 Web 页面提供的受限浏览、上传下载和浏览器访问权限控制行为。

### Modified Capabilities

无。

## Impact

- 影响新的桌面端、核心服务、本地 Web 页面和本地存储设计。
- 影响设备发现协议、配对流程、传输任务状态机和浏览器访问控制。
- 引入跨平台桌面框架、本地存储和传输抽象接口的技术选型约束。
- 为后续公网发现、NAT 打洞和中继传输预留扩展接口，但不在本次实现范围内。
