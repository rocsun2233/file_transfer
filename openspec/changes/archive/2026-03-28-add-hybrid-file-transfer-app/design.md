## Context

本变更从零定义一个新的跨平台局域网文件传输产品，不复用现有实现。目标平台为 Windows、Linux、macOS，交互形态采用桌面主应用加内置本地 Web 入口的混合模型。桌面应用承担设备发现、配对、传输任务和系统集成；浏览器入口服务未安装客户端的临时设备访问。

首版只实现局域网场景，但用户已明确要求架构上可演进到互联网穿透。因此设计不能把任务模型、信任模型或连接模型与局域网直连硬编码绑定。系统还需要覆盖断点续传、批量任务、历史记录和拖拽发送，这意味着传输流程必须以持久化任务会话为中心，而不是一次性 HTTP 上传下载。

## Goals / Non-Goals

**Goals:**
- 提供 Windows、Linux、macOS 一致的局域网文件传输体验
- 同时支持客户端到客户端传输和浏览器临时访问
- 通过首次确认或配对码建立设备信任关系
- 支持文件夹、批量任务、进度、历史记录和断点续传
- 将发现、连接和数据通道抽象为可替换适配器，为未来互联网穿透预留扩展点

**Non-Goals:**
- 本次不实现公网发现、NAT 打洞或中继服务
- 本次不实现账号系统、云同步或移动端原生应用
- 本次不实现复杂企业权限管理

## Decisions

### 1. 采用桌面壳层 + 本地核心服务 + 连接抽象层的三层结构

桌面层只负责 UI、拖拽发送、托盘和系统通知；本地核心服务负责发现、配对、任务、传输和 Web 服务；底层通过发现适配器与传输适配器屏蔽具体链路。

这样做可以把高频变化的界面层与高可靠的传输内核分离，也能在未来增加新的外壳形态或链路实现时保持任务逻辑稳定。

备选方案是让桌面应用直接承载全部协议逻辑。这种方式实现更快，但会导致 UI、网络和任务状态耦合，后续扩展互联网穿透时改动面过大，因此不采用。

### 2. 使用持久化任务会话作为传输中心模型

每次发送都创建任务 ID，并持久化文件清单、目录结构、分片进度、失败状态和冲突策略。断点续传、重试、批量任务和历史记录都围绕该任务模型展开。

这样可以使不同底层通道共享同一套任务状态机。未来从局域网直连扩展到公网打洞或中继时，上层无需重写任务管理逻辑。

备选方案是直接使用浏览器式上传下载请求或一次性 Socket 会话。这种方式不利于恢复、批量和链路切换，因此不采用。

### 3. 首版安全采用“默认可发现，首次连接需确认或配对码”

系统默认公开设备存在，但第一次建立信任关系必须由接收端确认或输入一次性配对码。信任建立后，同一设备后续访问可免确认，但用户可撤销信任。

这是在家庭或办公室局域网场景下兼顾易用性与防误连的折中方案。默认隐藏设备虽然更安全，但会显著增加使用门槛；默认完全开放则无法满足基本信任控制，因此不采用。

### 4. 浏览器入口只提供受限能力

本地 Web 页面支持设备访问验证、浏览允许暴露的内容、上传下载和查看最近任务，但不承担复杂任务编排、批量高级控制或系统设置。

这样可以保持 Web 入口轻量且稳定，避免把桌面主体验降级为网页实现。复杂交互继续由桌面端承载。

### 5. 为未来互联网穿透定义链路升级序列

连接流程固定为：优先局域网发现和直连；若目标设备不在同一局域网，再接入云端目录；随后尝试 NAT 打洞；打洞失败则回退到中继。

首版不实现这些公网能力，但传输层接口和状态模型从一开始就需要允许“直连失败后降级到另一适配器”的切换。

## Risks / Trade-offs

- [混合形态增加系统边界复杂度] → 通过明确区分桌面能力与 Web 受限能力，避免双端职责混乱。
- [跨平台设备发现兼容性差异] → 同时保留自动发现和手动输入地址，降低单一发现机制失效风险。
- [断点续传与批量任务会扩大状态管理复杂度] → 采用持久化任务会话与统一状态机，避免把恢复逻辑散落在多个传输入口。
- [未来互联网穿透需求可能改变接口] → 提前用适配器抽象发现与传输层，避免业务层直接依赖 LAN 实现。
- [浏览器入口可能带来额外暴露面] → 未配对访问默认进入受限入口，并要求验证码或桌面端确认。

## Migration Plan

这是新产品定义，无存量系统需要迁移。

实施顺序建议为：

1. 建立桌面壳层与本地核心服务的基础工程
2. 实现任务模型、本地存储和状态机
3. 实现局域网发现、配对与信任关系
4. 实现文件与文件夹传输、断点续传和批量任务
5. 实现本地 Web 页面与受限访问控制
6. 增加跨平台集成测试和兼容性验证

若某一阶段存在重大稳定性问题，可先停留在纯局域网直连能力，不影响整体架构。

## Open Questions

- 首版是否需要对大文件传输增加显式带宽限制或后台节流策略。
- 浏览器入口默认暴露的目录范围是否应限制为单独的共享空间，而不是完整接收目录。
- 首版是否需要支持单次分享码的有效期配置。
