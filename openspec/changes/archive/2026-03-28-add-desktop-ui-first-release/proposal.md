## Why

当前程序已经具备设备发现、配对、真实 TCP 传输和任务状态管理，但桌面端仍停留在最小原型，用户无法通过完整图形界面顺畅地发起传输、处理接收确认、查看任务进度、查看历史记录和调整关键设置。现在需要补齐桌面主界面，使这些已有能力真正变成可用的桌面应用体验。

## What Changes

- 新增完整首版桌面工作台界面，覆盖设备列表、任务面板、历史记录和设置。
- 新增图形界面的发送入口，支持选择文件、选择文件夹和拖拽发送入口。
- 新增接收确认与拒绝对话框，并允许任务级冲突策略覆盖。
- 新增可恢复失败任务的重试入口。
- 新增桌面状态映射层，把核心服务状态整理成 UI 可消费模型。

## Capabilities

### New Capabilities
- `desktop-transfer-workbench`: 定义桌面端如何展示设备、任务、历史记录、设置以及发送/确认/重试等交互行为。

### Modified Capabilities

无。

## Impact

- 影响 `hybrid_transfer/desktop.py` 及其后续拆分出的桌面 UI 模块。
- 影响桌面端如何消费任务状态、信任状态、冲突策略和共享目录配置。
- 影响测试，需要新增桌面状态测试、交互测试和接入后的回归测试。
